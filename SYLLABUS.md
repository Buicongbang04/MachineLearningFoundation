# Syllabus — 12-Week Plan

Each week pairs reading from `docs/` with a runnable artifact in `notebooks/`, `labs/`, `assignments/`, or `projects/`. Times are guidelines — most learners spend 4-6 hours per week.

| Week | Topic                                          | Deliverable                |
|------|------------------------------------------------|----------------------------|
| 1    | Introduction to AI/ML/DL, Python, NumPy        | Notebook warm-up           |
| 2    | Linear algebra for ML                          | Assignment 01              |
| 3    | Probability, gradient, optimization            | Gradient-descent demo      |
| 4    | ML pipeline, train/val/test, metrics           | Lab pipeline               |
| 5    | Linear regression, overfitting                 | Project — regression       |
| 6    | KNN, classification metrics                    | Project — Iris             |
| 7    | K-means, clustering                            | Customer segmentation      |
| 8    | Naive Bayes, text classification               | Spam classifier            |
| 9    | Logistic / Softmax regression                  | Multi-class classifier     |
| 10   | Neural networks (intro)                        | MLP notebook               |
| 11   | PCA, SVM, model comparison                     | Model benchmark            |
| 12   | Final-project presentation                     | Final report               |

## Weekly Loop

For each week:

1. Read the corresponding `docs/NN_*.md` notes.
2. Open and run the companion notebook in `notebooks/`.
3. Tweak hyperparameters and observe what changes.
4. Solve the assignment / lab listed in the deliverable column.
5. Write a one-paragraph reflection (what surprised you, what's still fuzzy).

## Checkpoints

- **After Week 4** — you should be able to set up a clean train/val/test split and train a baseline.
- **After Week 7** — you should be comfortable comparing two models on the same metric.
- **After Week 10** — you should know how to read a learning curve and diagnose overfitting.
- **After Week 12** — you should have a polished end-to-end mini-project in `projects/`.

## Pace adjustments

- **Faster** (2x speed): pair two weeks per week, drop "Reading" days, skip optional further-reading lists. Suitable for graduate students already familiar with Python and math.
- **Slower** (0.5x speed): take two weeks per row above, and spend extra time on the math review (weeks 2-3). Recommended for absolute beginners.
