# Contributing

This guide is for anyone extending the course — adding a chapter, a lab, an assignment, or a mini-project.

## TL;DR

- Don't hand-write `.ipynb` files. Write a builder script in `src/build_*.py` and check that in alongside the produced notebook. Notebooks are diff-noisy; builders are not.
- Every new notebook / project / lab should be executable end-to-end on a clean `aicourse` env: `jupyter nbconvert --to notebook --execute <path>` must finish without errors.
- Generated artifacts (model outputs, downloaded datasets, run logs) belong in `.gitignore`, not in the repo. Keep `.gitkeep` files so empty output folders are tracked.

## Repository layout

```
.
├── README.md, SYLLABUS.md, ROADMAP.md   # learner-facing entry points
├── docs/                                # narrative chapter notes (Markdown)
├── notebooks/                           # one runnable notebook per chapter
├── labs/                                # multi-chapter integration labs
├── assignments/                         # student-graded exercises
├── projects/                            # mini-projects + FINAL_PROJECT_TEMPLATE
├── datasets/                            # docs only; raw files are gitignored
├── src/                                 # builder scripts + shared utilities
├── figures/                             # generated diagrams shared across chapters
└── reports/                             # optional written experiment reports
```

## Adding a new chapter

1. **Doc.** Copy `docs/CHAPTER_TEMPLATE.md` to `docs/NN_topic.md` and write the narrative.
2. **Notebook builder.** Create `src/build_chapter_NN_*.py` following the pattern in `src/build_chapter_15_notebook.py`:
   - Use `nbformat.v4` and the `md` / `code` helpers.
   - End with `write_notebook(cells, ROOT / "notebooks" / "chapter_NN_topic.ipynb")`.
3. **Build & verify.**
   ```bash
   conda run -n aicourse python src/build_chapter_NN_topic.py
   conda run -n aicourse jupyter nbconvert --to notebook --execute notebooks/chapter_NN_topic.ipynb --output chapter_NN_topic.ipynb
   ```
4. **ROADMAP.** Add a row for the new chapter and flip its status when complete.
5. **Commit.** Include the builder, the executed notebook, and any new figures under `figures/` in the same commit.

## Adding a new project

1. Copy `projects/FINAL_PROJECT_TEMPLATE/` to `projects/<project_name>/`.
2. Fill in `README.md`, `report.md`, and the three `src/*.py` files.
3. The notebook in `notebooks/<project>.ipynb` should be built from a script (see `projects/project_01_house_price_regression/build_notebook.py` for an example) and re-use the project's own `src/` helpers — so `python src/train.py` and the notebook produce identical metrics.
4. Save figures via `fig.savefig(PROJECT_DIR / "figures" / "<name>.png", dpi=120)`. The `figures/` folder ships a `.gitkeep`; generated `.png` files are *not* committed.
5. Save metrics via `json.dump(results, RESULTS_DIR / "metrics.json")` — gitignored, regenerated by `train.py`.

The fully worked example is `projects/sample_final_project_wine/`:

```
sample_final_project_wine/
├── README.md, report.md, requirements.txt
├── src/{data_preprocessing.py, train.py, evaluate.py}
├── notebooks/sample_final_project.ipynb   # built from src/build_sample_final_project_notebook.py
├── figures/.gitkeep                        # generated .png files land here, not tracked
└── results/.gitkeep                        # metrics.json lands here, not tracked
```

## Style

- **Code.** Black-ish formatting (4-space indent, no trailing whitespace). Type hints encouraged on `src/` helpers; not required inside notebooks.
- **Notebooks.** One concept per cell. Markdown cells should explain the *why* of the next code cell. No silent re-imports across cells.
- **Comments.** Prefer well-named variables over comments. Comment only when the *why* is non-obvious.
- **Tests.** None required for course material. The execution check (`nbconvert --execute`) is the de-facto test.

## .gitignore policy

Anything generated by running code is gitignored. Concretely:

- `projects/*/results/*.json` and `projects/*/results/*.csv` — outputs of `train.py`.
- `projects/*/figures/*.png` — plots saved by notebooks. *(Today these are produced fresh per run; if a figure is conceptually static and belongs in the docs, copy it to `figures/` at the repo root and commit it there.)*
- `datasets/*.csv`, `datasets/*.parquet`, etc. — keep only `datasets/README.md` describing where the data comes from.
- `book_ML.pdf`, `Instruction.pdf` — copyrighted PDFs shared by the instructor; never commit.

When you add a new producer (a notebook or script that writes to disk), add the matching pattern to `.gitignore` in the same commit.

## Running everything locally

```bash
conda env create -f environment.yml
conda activate aicourse

# Build any notebook from its source script.
python src/build_chapter_15_notebook.py

# Re-execute (this is also the test).
jupyter nbconvert --to notebook --execute notebooks/chapter_15_svm.ipynb --output chapter_15_svm.ipynb

# Train a mini-project from CLI.
cd projects/sample_final_project_wine
python src/train.py
```

## Pull-request checklist

- [ ] New / modified notebooks execute cleanly with `jupyter nbconvert --execute`.
- [ ] Builder scripts in `src/` produce the same notebook bit-for-bit (don't hand-edit the `.ipynb`).
- [ ] New disk writes have matching entries in `.gitignore`.
- [ ] `ROADMAP.md` reflects the new content's status.
- [ ] No `book_ML.pdf`, `Instruction.pdf`, or large dataset files staged.
