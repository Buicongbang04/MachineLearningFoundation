# Course Overview

**Intro Machine Learning for AI** is an open, hands-on curriculum that turns the Vietnamese textbook *Machine Learning cơ bản* (Vũ Hữu Tiệp) into an AI-Foundation course.

This document covers the *why* of the course — audience, prerequisites, and design philosophy. For the *what* and *how*, see:

- `README.md` — install instructions, repo layout, full chapter list.
- `SYLLABUS.md` — a 12-week schedule with deliverables.
- `ROADMAP.md` — chapter-by-chapter goals, topics, outcomes, and artifacts.

## Who this is for

- CS, data-science, and software-engineering students starting in AI.
- Self-learners who know basic Python and want a structured path.
- Deep-learning learners who lack a solid classical ML and math foundation.
- Anyone building an open-source course or teaching repository.
- Aspiring AI Engineers, Data Scientists, or ML Engineers.

## Prerequisites

- Python basics (variables, lists, dictionaries, loops, classes).
- Jupyter Notebook or Google Colab.
- High-school math (functions, derivatives, vectors).
- Ability to read and run Python code.

Heavier math (linear algebra, probability, gradient, optimization) is revisited from scratch in the early chapters.

## Design Philosophy

- **Start simple.** Begin with linear models. Understand *why* a model works, *where* it breaks, and *how* to improve it before stacking complexity.
- **Intuition first, formulas second.** Each algorithm is motivated in plain language before any math is shown.
- **Code goes with theory.** Every chapter ships with a runnable notebook. You should change parameters, watch results, and explain what happened.
- **Evaluation is mandatory.** This course does not stop at `model.fit()`. You will compare against a baseline, analyze errors, and detect overfitting.
- **Reusable repo.** Not a pile of stray notebooks — chapter, lab, exercise, dataset, figure, report, template, and final project all live in well-defined folders.

## What you will be able to do

- Explain how Machine Learning differs from traditional programming.
- Recognize when a problem is regression, classification, clustering, or something else.
- Tell when ML is the right tool — and when it is not.
- Frame an ML task in terms of data, model, loss, and metric.
- Train and evaluate models with scikit-learn correctly (no train-set evaluation).
- Build an end-to-end pipeline that someone else can reproduce.

## Beyond this course

The repo is intentionally an "AI Foundation Repository." Once you finish it, you have the base to continue with:

- Deep Learning with PyTorch.
- Computer Vision.
- Natural Language Processing.
- Large Language Models.
- Basic MLOps.
- AI Engineering and real product development.
